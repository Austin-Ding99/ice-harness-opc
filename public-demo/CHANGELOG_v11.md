# Ice-Harness OPC v11 改进说明

## 版本定位

v11 在 v10 的实时数据穿刺、云端切片、标注和 PPTX 报告链路基础上，重点补强评审中容易被追问的生产化能力：任务日志持久化、失败重试、断点续传和 Agent 健康状态。

本版本目标不是增加表面功能，而是让 Harness 架构从“可演示的 Agent 矩阵”进一步变成“可追踪、可恢复、可解释的 Agent 运行系统”。

## 核心改进

### 1. 新增监理 Agent 运行时

新增 `agent_supervisor.js`，统一管理运行态：

- 持久化任务台账：写入 `localStorage`，刷新页面后仍可查看近期任务。
- 持久化运行日志：将关键 Agent 日志同步沉淀为结构化记录。
- Agent 健康状态：展示在线、运行中、待命、异常和失败次数。
- 断点快照：保存当前数据源、通道矩阵、已加载行数、监控阈值、异常状态、通道显隐状态和播放速度。
- 断点恢复：可恢复到上次快照的实验进度，恢复后保持暂停，等待超级个体确认继续。

### 2. 主控台新增可视化监理看板

主页面右侧 Harness 状态看板新增三类内容：

- 断点操作：恢复断点、保存断点、清空台账。
- Agent 健康卡片：每个核心 Agent 显示最近心跳、状态、失败次数和最新动作。
- 持久化任务台账：展示最近任务、执行状态、所属 Agent、更新时间和重试次数。

这部分是路演时最直接的生产化证据，能支撑“监理Agent负责全局状态监控”的方案表述。

### 2.1 优化环境异常提示

修复数据成功接入后偶发显示“环境异常: Script error.”的问题：

- 为外部 CDN 脚本增加 `crossorigin="anonymous"`，便于浏览器返回更明确的错误信息。
- 全局错误监听不再把第三方库的无明细 `Script error.` 或浏览器 ResizeObserver 提示当作致命故障。
- 真正的一方运行错误仍会显示在数据状态栏，并写入监理 Agent 台账。

### 2.2 优化监理看板布局

- Agent 健康状态卡片全部展开显示，不再在健康区内部出现滚动条。
- 下方任务/触发事件台账增加滚动条，适合持续追踪更多历史事件。
- 健康卡片中的“最近心跳”文案调整为“最近动作”，更贴近现场说明语境。
- Agent 状态看板迁移到自然语言输入框下方、通道矩阵和监控视图上方，横向固定展示；右侧面板恢复为实时触发日志主视图。
- 修复监理 Agent 断点快照属性与 `checkpoint()` 方法重名导致“checkpoint is not a function”的问题，开始监控与 NL2Action 启动链路恢复正常。

### 3. 数据接入链路任务化

`agent_extractor.js` 接入监理运行时：

- CSV/TSV 导入时自动登记“接入实验数据”任务。
- 表头定位失败、文件行数不足、有效数据为空会记录失败状态。
- 成功解析后记录识别通道数和载入行数。
- 数据接入完成后自动保存断点，形成可恢复起点。

### 4. 实验监控链路支持自动快照

`agent_monitor.js` 接入断点机制：

- 监控启动时保存快照。
- 每滚动 25 行自动保存一次快照。
- 一次性加载全部数据后保存快照。
- 实验完成或数据流自动结束后保存快照。

这样即使页面刷新或演示中断，也可以恢复到接近中断前的实验状态。

### 5. 云端切片同步支持失败重试

`agent_cleaner.js` 将切片同步封装为监理任务：

- 框选完成后创建“同步切片到云端Sheet”任务。
- 同步过程由 `runWithRetry` 执行，默认最多重试 2 次。
- 成功后保存“切片同步完成”断点并打开云端 Sheet 页面。
- 失败会记录到任务台账和 Agent 健康状态。

### 6. 报告生成任务回写主控台

`cloud_sheet.html` 新增报告任务回写逻辑：

- PPTX 生成开始时写入任务状态。
- DeepSeek 报告分析请求加入重试封装。
- 成功生成 PPTX 后回写完成状态和文件名。
- 失败时回写失败状态，并通过跨页日志桥同步到主控台。

### 7. v11 交互修正与 AI 表头识别增强

- “一次性加载全部数据”现在会自动进入实验完成状态，并输出“试验完成。右上角【数据分析】已就绪，可框选感兴趣数据段并生成报告。”完成按钮会同步禁用。
- 云端 Sheet 的曲线 tooltip 改为主题自适应弹窗，并优先显示在鼠标左侧，减少遮挡当前数据。
- 修复云端 Sheet 红色虚线标记拖动范围与图线坐标不一致的问题，使用 Chart.js 原生坐标约束，避免拖到图线区域外或无法触达右侧图区。
- 暗色模式下关键标记标题色改为更亮的红色，提高“① 时间：...”的辨识度。
- 标记数据弹窗最小高度改为可显示 1 行数据。
- 每个 Agent 健康卡片新增“查看日志”按钮，可弹窗查看该 Agent 的时间戳运行日志、触发代码引用和任务历史。
- CSV/TSV 表头识别升级为 Function Calling：上传文件后读取前 10 行，由表格清洗 Agent 判断表头行、时间列、温度列和功率列；无 API Key 或识别失败时自动回退本地规则，保证演示稳定。
- 修复坏点剔除后实验监控 Agent 健康状态仍停留在“异常”的问题：清洗成功后会写入“预警闭环解除”日志、恢复在线状态，并清零该 Agent 的失败计数；“异常拦截”作为正常预警行为不再累计为失败次数。
- 修复云端 Sheet 打开后自动出现历史虚线标记的问题：新切片同步会清理旧标记缓存，云端页默认不再自动恢复旧标记，只有点击曲线后才生成新的虚线与序号。
- “清空台账”现在会同步清理 Agent 健康状态缓存和异常计数，刷新 `index.html` 后不再继承旧的异常次数；界面文案从“失败”改为“异常计数”，避免和任务失败混淆。
- 云端 Sheet 标记弹窗刷新时不再重建缩放手柄 DOM，避免出现鼠标指针正确但拖拽无响应的情况。
- PPTX 生成前会净化大模型返回文本中的 Office XML 非法控制字符，并等待 `writeFile` 完成后再回写任务完成状态，降低 PowerPoint 提示修复或受保护视图无法读取的概率。
- “恢复断点”日志文案调整为“恢复到指定行并暂停，点击【开始监控】继续”，并增加短时间重复点击防抖，减少连续点击导致的重复恢复日志。
- 主控台温度图与功率图横坐标改为短格式 `HHMMSS`，减少长时间戳挤占导致的上下图 0 点错位；完整时间仍保存在独立缓存中，鼠标悬浮数据弹窗和云端切片数据继续显示完整时间信息。

## 演示建议

推荐 v11 路演顺序：

1. 导入实验 CSV/TSV，展示任务台账出现“接入实验数据”。
2. 点击开始监控，展示实验监控 Agent 进入运行中。
3. 触发或等待异常点，展示预警日志和健康状态变化。
4. 点击保存断点，再刷新页面或重新打开页面，点击恢复断点。
5. 框选感兴趣数据段，展示切片同步任务和云端 Sheet。
6. 在云端 Sheet 标注关键点并生成 PPTX，展示报告任务回写。

## 当前边界

- v11 的持久化基于浏览器 `localStorage`，适合赛事 DEMO 和单机验证；正式生产版可替换为后台数据库或企业云文档 API。
- Agent 健康状态基于前端事件心跳，能体现运行机制；生产版可扩展为 WebSocket 心跳、任务队列状态和后端守护进程。
- 失败重试已覆盖云端切片同步、NL2Action 路由和报告生成请求；后续可继续覆盖真实云表格 API、文件上传和后台数据流连接。

## 交付文件

- `index.html`：主控台与 Harness 协同状态看板。
- `cloud_sheet.html`：云端切片、标注、CSV 下载和 PPTX 报告生成。
- `agent_extractor.js`：数据提取与通道选择 Agent。
- `agent_monitor.js`：实验监控与预警 Agent。
- `agent_cleaner.js`：表格清洗与云端同步 Agent。
- `agent_report.js`：分析与报告 Agent 兼容脚本。
- `agent_supervisor.js`：v11 新增监理 Agent 运行时。
